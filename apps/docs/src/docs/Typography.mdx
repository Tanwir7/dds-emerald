# Typography

## System Rules

- Display headings use <code className="dds-docs-token-code">{"var(--dds-font-display)"}</code>.
- Body text uses <code className="dds-docs-token-code">{"var(--dds-font-sans)"}</code>.
- Code and data use <code className="dds-docs-token-code">{"var(--dds-font-mono)"}</code>.
- Component SCSS must use typography tokens, never hardcoded font-family strings.
- Body-level text uses <code className="dds-docs-token-code">{"font-feature-settings: \"cv02\", \"cv03\", \"cv04\", \"cv11\""}</code> globally.
- Numeric and tabular data uses <code className="dds-docs-token-code">{"font-variant-numeric: tabular-nums"}</code>.
- DM Sans optical sizing is handled by loaded font files and font-size changes. Do not set component-level font variation settings.

## Component Usage

Use <a href="?path=/docs/core-components-text--docs">Core Components/Text</a> for body copy, labels, inline text, and muted supporting copy. Use <a href="?path=/docs/core-components-heading--docs">Core Components/Heading</a> for semantic headings and page hierarchy. Keep semantic heading order separate from visual size: choose the correct heading element first, then choose the visual size.
