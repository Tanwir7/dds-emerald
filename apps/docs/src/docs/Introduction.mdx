# Emerald Design System

Emerald is the DDS design system for building accessible, consistent product interfaces. It provides shared tokens, React components, documentation, and implementation rules for Digital Dev Studio products.

## What Emerald Includes

- Foundations for tokens, color, and typography
- Core React components with Storybook docs and examples
- Accessibility requirements for component behaviour and documentation
- Generated token documentation sourced from `packages/tokens/src/tokens.css`

## Core Rules

- CSS custom properties, SCSS variables, and class names use the `dds` prefix.
- Components use SCSS modules, not Tailwind CSS or inline styles.
- Components consume semantic Tier 2 tokens wherever possible.
- React components use `React.forwardRef`, accept `className`, and use `clsx`.
- Interactive accessibility primitives should use Radix UI when an appropriate primitive exists.
- UI component border radius is `var(--dds-radius-none)` unless a documented Emerald exception allows `var(--dds-radius-full)`.
- Accessibility is part of done: keyboard support, screen reader expectations, focus management, and axe checks are required for component work.

## Where To Start

Start with `Foundations/Tokens` to understand the token model, then review `Foundations/Colors` and `Foundations/Typography`. For implementation examples, move into `Core Components` and inspect the component docs, controls, and source examples.
